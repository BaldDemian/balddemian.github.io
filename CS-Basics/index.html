计算机基础知识备忘

¶硬件与体系结构

¶冯诺依曼架构与哈佛架构

  • 冯诺依曼架构：数据和程序存放在一个存储空间中，统一编址，指令和数据通过同一条总线访问
  • 哈佛架构：数据和程序存放在不同的存储空间中，即程序存储器和数据存储器是两个不同的存储器，每个存储器独立编址、独立访问。与之相对应的是系统中设置的两条总线：程序总线与数据总线，从而使得数据（包括指令）的吞吐量大了一倍

¶位数、字长

现在最常用的 CPU 都是 64 位。64 位的 CPU 指的是：机器字长是 64 位。那么机器字长又是什么呢？机器字长，即计算机进行一次整数运算所最多能处理的二进制数据的位数。我们也许还会看到这几个名词：指令字长、存储字长、数据字长。哇这都是些啥啊，要被绕昏了，且看：

  1. 存储字长：主存（即内存）中的每个存储单元所能存储的二进制数据的位数称为存储字长。需要注意的是存储字长和地址空间也不是一个东西。地址空间是指能有多少个存储单元，存储字长是单个存储单元中存储的二进制代码的长度。编址方式就决定了存储字长，比如按字节编址，就是说一个存储单元中有 1 个字节，存储字长是 1 个字节；地址空间是由地址码长度决定的
  2. 指令字长：我们知道一条指令由操作码和地址码组成，一条指令的长度就是指令字长。同一个指令集中，不同指令的字长可以不同
  3. 数据总线的宽度叫做数据字长，数据总线的宽度与 CPU 控制器中的 MDR 寄存器的位数相同。数据字长可以是存储字长的整数倍，意味着 CPU 一次访存可以同时访问多个存储单元
  4. 早期的计算机，存储字长一般与指令字长、数据字长相等。但是现代计算机中可以不一样，例如指令字长是存储字长的两倍，那我们称这条指令是一条双字指令，它占据内存中两个连续的存储单元，那么则需要两次访存才能取出一条完整的指令（即取指周期是机器周期的两倍。什么是机器周期？我们暂且认为机器周期就是主存一个存储单元的存取周期）
  5. 操作系统的字长指的是操作系统可寻址的位数。64 位的操作系统是为 64 位的架构的计算机设计的操作系统，32 位的操作系统也可以用于 64 位架构的机器，但是 64 位的操作系统不可以运行在 32 位架构的计算机上

¶硬堆栈与软堆栈

堆栈的实现方式有两种：

  • 硬堆栈：用寄存器组实现堆栈，优点是速度快，缺点是成本高、容量小
  • 软堆栈：用内存实现堆栈，优点是容量大、成本低

不管是哪种实现方式，均有一个 SP 寄存器（stack pointer）指向当前的栈顶元素的地址（或者下一个地址）

¶RISC 与 CISC

  • RISC, Reduced Instrcution Set Computer。通过少量的简单指令来组合出复杂的功能。RISC 的出现与计算机中的「二八法则」有关[1]，即 80% 的功能只用到了 20% 的指令；

    电路简单（？），并且由于每个指令都不复杂，使得它们的执行时间差不多（几乎都可以在一个时钟周期中完成），一定使用指令流水线技术；

    仅有 LOAD 与 STORE 指令可以访存，因此 RISC 中的寄存器较多，因为必须先将数据从主存中 LOAD 进寄存器中；

    可以进行编译优化；

    控制器采用「硬布线控制器」

    代表：ARM 架构

  • CISC, Complex Instruction Set Computer。为复杂的功能单独设计指令；

    可以访存的指令有很多；如某些运算指令可以直接从主存读取数据进行运算，不需要 LOAD 指令

    难以进行编译优化；

    控制器采用「微程序控制器」

    代表：PC 上的 x86 架构

¶GPU

GPU 是 Graphics Processing Unit 的简称。

CPU 追求低延时(latency)，而 GPU追求 高吞吐(throuputs)。

GPU 有着更多的 ALU，因此 GPU 具有强大的并行计算能力，如矩阵的计算。因此，GPU 常用于加速深度学习。

GPU 有自己的 RAM，即 VRAM(video ram)。VRAM 比 CPU 中的 RAM 容量小。

GPU 在处理控制逻辑（分支、跳转、循环）较多的运算时不如 CPU。

¶大端序、小端序

现代计算机主存一般采用按字节编址，即一个存储单元存储 1 个字节（或者理解为 8 个比特共享一个地址）。但是对于那些占据多个字节的数据类型，应该如何安排不同字节在内存中的排列方式呢？我们有两种解决方案：大端序和小端序

口诀：小端序：高字节高地址；大端序：高字节低地址

指的是多字节的哪一端存储在该值的起始地址处。

大端序中，最高有效字节 MSB 存储在地址的最低位，最低有效字节 LSB 存储在地址的最高位；小端序则与大端序相反。具体看下面示例：

我们还需要指出什么是高字节：在 9877 的 16 位补码形式中，0010 0110 就是高字节。

为了方便演示，我们规定内存的地址从上到下增加，那么 9877 在大端序（低地址高字节）的机器中是这样存储的：

¶热插拔 热加载 热部署

  • 热插拔

    热插拔 Hot Swap 即带电插拔，指的是在不关闭系统电源的情况下，将模块、板卡插入或拔出系统而不影响系统的正常工作，从而提高了系统的可靠性、快速维修性、冗余性和对灾难的及时恢复能力等。对于大功率模块化电源系统而言，热插拔技术可在维持整个电源系统电压的情况下，更换发生故障的电源模块，并保证模块化电源系统中其他电源模块正常运作。（摘自百度百科）

  • 热加载

    指的是在不重启服务的情况下，使得修改的程序代码生效。典型的热加载的案例是 Vue，比如新加了一个页面，可以不必重新运行 npm build 和 npm serve 即可查看网页变更

  • 热部署

    与以上两个概念相似，指为软件增加新模块后，可以不必重启整个系统。对于某些重要系统而言，重启关机是不可接受的，所以热部署很重要

¶从按下开机键到操作系统启动发生了什么？

老八股了

主要基于 X86 架构

  • 通电后激活 CPU，IR(Intruction Register) 置为 BIOS 的第一条指令(存储在 CPU 内部的 ROM 中，包括指令集的微指令序列也存储在这个 ROM 中)，CPU 开始执行 BIOS 程序

  • BIOS 执行 POST 自检，即 Power-On Self Test

  • 一台机器中可能有多个磁盘，BIOS 根据保存在 CMOS（主板上的另一个区域，保存主板的重要参数，如电压、系统时间）的磁盘启动顺序，或与用户交互的顺序，启动第一位的磁盘

  • BIOS 搜寻该磁盘第 0 盘面第 0 磁道第 1 扇区，如果该扇区以 0xAA55（一个特定的魔数） 结尾，则被 BIOS 视为主引导扇区

    主引导扇区也叫 MBR，Master Boot Record，其内容是一段二进制代码、磁盘分区表、魔数

    磁盘分区表是什么？磁盘分区(partition)是什么？

    磁盘分区就是将磁盘分为一个或多个柱面（柱面：不同盘面上的同一位置的扇区构成一个柱面，想象一下圆柱的形状~），每个分区可以使用不同的文件系统。比如 Windows 下经典的 C 盘和 D 盘就是两个分区。那么操作系统怎么知道每个分区开始于哪个柱面、占据几个柱面呢？每个分区的起始扇区和大小被记录在 MBR 的分区表中。

  • 此时将这个扇区装入物理地址 07C00H 处，修改 PC 为 07C00H，BIOS 交出 CPU，开始执行 BootLoader。前面提到过，MBR 中的可执行代码很短，其唯一的作用是将第二级的 BootLoader 加载进内存，并开始执行

  • 第二级的 BootLoader 可以让用户选择要加载的操作系统。将对应的操作系统内核加载进内存后，交由内核执行更多的工作…

再补充一个有意思的细节

  • 我们知道读取键盘输入、在屏幕上显示信息，是通过中断实现的，在汇编编程中，我们调用操作系统提供的中断。但是在二阶段的引导操作系统时，屏幕上已经出现了输入，并且可以读取键盘输入（让用户选择要启动哪个操作系统），这同样是通过中断实现的。这里的中断是由 BIOS 提供的。也就是说，存在两套中断，一套由 BIOS 提供，一套由操作系统提供

¶格雷码

在一个整数的二进制表示的序列中，如果前一个数与后一个数仅有一个二进制位不同，且首尾元素也仅有一个二进制位不同，那么这个序列中的编码就是格雷编码。如序列 $0, 1, 3, 2$，其对应的二进制序列为 00, 01, 11, 10。

$n$ 位的格雷码序列，即每个二进制数均为 $n$ 位，那么对应的十进制数有 $2^n$ 个，即 $[0, 2^n-1]$。在该序列第 $i$ 个位置上的格雷码，计算方式如下：$gray[i]=i \oplus (i>>1) $

¶布隆过滤器

在平常的算法题中，判断元素是否存在在某个集合中，我们常用的是哈希表。但是在千万级的数据量时，使用哈希表显然是不合适的。布隆过滤器可以节省空间，并且插入元素和查询元素的速度都很快，缺点是有一定概率误判：如果一个元素经布隆过滤器判断后存在，那么它也有可能不存在（由于哈希碰撞的原因）；但是一个元素经布隆过滤器判断后不存在，那么它就一定不存在。

从上述描述中，我们发现布隆过滤器的效率取决于哈希碰撞的概率。那相比一般的 HashTable，布隆过滤器是如何降低哈希冲突的概率呢？

布隆过滤器实现的简述：

  • 一个 BitSet（初始时所有位置均是 0）
  • n 个哈希函数

我们对待加入的元素 x 执行 n 次不同的散列，得到 n 个的结果，我们将这 n 个结果对应的 BitSet 下标处置 1，这就完成了元素的插入；

当查询元素是否存在时，我们对待查元素 x 同样执行 n 次不同的散列，得到 n 个 BitSet 中的下标，然后去检查这 n 个下标处是不是均为 1；若均为 1，那说明待查元素 x 「可能」存在在集合中（虽然布隆过滤器的算法导致哈希碰撞的概率较低，但毕竟不是不可能事件）；若有一处不为 1，那么 x 「肯定不在」集合中。

同时相比于一般的哈希表，由于布隆过滤器中存储的不是实际的数据，数据的安全性也提高了

¶Java Web 开发中的一些基本概念

¶Servlet 与 JSP

  • Servlet 是一种用于创建 Web 应用的技术，用于处理服务器收到的请求。大致的过程时，服务器收到请求后，发送给对应的 Servlet
  • Servlet 是一套 API，一个 Java 类
  • Servlet 容器是运行在服务器之上的，用来管理 servlets
  • JSP 即 Java Server Pages。简单来说，即基于 Java 的 HTML。在经典的后端 MVC 分层中，可以认为 JSP 在 view 层，Servlet 在 controller 层
  • 使用 JSP 的 Web 应用开发属于前后端不分离，正逐步被前后端分离的范式所取代。因此，JSP 逐渐被淘汰；而 Servlet 仍是 Java Web 开发的基石，著名的 Spring 框架的底层仍是 Servlet。可以认为，在前后端分离的开发中，MVC 变成了 MC，没有了 view

¶JAR 与 WAR

  • JAR 即 Java Archive。是一个压缩文件，存储字节码文件和编译好的库，及 JAR 包的元信息

  • WAR 即 Web Application Resource 或 Web Archive。其中存储部署在 Servlet/JSP 容器中的网络应用，比如静态的 HTML、CSS、JS 文件

  • 关键区别

    JAR 的用途更加广泛，可以将其用作库或者是应用（比如想不借助 Maven 使用第三方的库，可以直接下载对应 JAR 包导入工程路径下）；而 WAR 仅能用于 Web 应用。

    此外，可执行的 JAR 可以直接通过命令行运行，而所有的 WAR 必须在服务器上运行
