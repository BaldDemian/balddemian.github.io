¶64 位的处理器 与 64 位的操作系统

64 位的 CPU 指的是：机器字长是 64 位。那么机器字长又是什么呢？机器字长，即计算机进行一次整数运算所最多能处理的二进制数据的位数

  1. CPU 中通用寄存器的宽度为 64 位
  2. CPU 中数据总线的宽度为 64 位
  3. CPU 的指令集中的指令字长（即操作码 + 操作数的长度）最高为 64 位的（因为指令也是数据。但是指令字长本身可以比 CPU 的位数短，高位补 0 即可）
  4. 主存（即内存）中的每个存储单元所能存储的二进制数据的位数称为存储字长。
  5. 指令的长度（操作码 + 操作地址）称为指令字长
  6. 早期的计算机，存储字长一般与指令字长、数据字长相等。但是现代计算机中可以不一样，例如指令字长是存储字长的两倍，那么则需要两次访存才能取出一条完整的指令（即取值周期是机器周期的两倍）
  7. 64 位的操作系统是为 64 位的架构的计算机设计的操作系统，32 位的操作系统也可以用于 64 位架构的机器，但是 64 位的操作系统不可以运行在 32 位架构的计算机上（以上解释基于王道计算机组成原理考研复习指导）

¶大端序、小端序

Big-Endian and Little-Endian，指的是多字节的哪一端存储在该值的起始地址处。

大端序中，高位字节存储在地址的最低位，最低字节在地址的最高位；小端序则与大端序相反。具体看下面实例：

我们观察十进制数 9877，其原码为 10011010010101，补码为 0010 0110 1001 0101（补 0 扩展为 2 个字节，计算机内部存储的是数的补码、补码原码的转换等不再赘述）。

我们还需要指出什么是高字节：在 9877 的 16 位补码形式中，0010 0110 就是高字节。

为了方便演示，我们规定内存的地址从上到下增加，那么 9877 在大端序（低地址高字节）的机器中是这样存储的：

00100110

10010101

小端序（低地址低字节）是这样的：

10010101

00100110；

大端序用于 PowerPC、SPARC 等处理器，JVM 读取 .class 文件也是按照大端序的；

小端序用于 x86 处理器；

¶热插拔

热插拔 Hot Swap 即带电插拔，指的是在不关闭系统电源的情况下，将模块、板卡插入或拔出系统而不影响系统的正常工作，从而提高了系统的可靠性、快速维修性、冗余性和对灾难的及时恢复能力等。对于大功率模块化电源系统而言，热插拔技术可在维持整个电源系统电压的情况下，更换发生故障的电源模块，并保证模块化电源系统中其他电源模块正常运作。（摘自百度百科）

¶热加载

指的是在不重启服务的情况下，使得修改的程序代码生效。典型的热加载的案例是 Vue，比如新加了一个页面，可以不必重新运行 npm build 和 npm serve 即可查看网页变更

¶热部署

与以上两个概念相似，指为软件增加新模块后，可以不必重启整个系统。对于某些重要系统而言，重启关机是不可接受的，所以热部署很重要

¶停机问题

停机问题，halting problem，离散数学中的一个经典问题。通俗地说，停机问题就是判断任意一个程序是否能在有限的时间之内结束运行的问题。（摘自维基百科）[1]
